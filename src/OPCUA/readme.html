OPC UA Demo Project Readme

Overview

This LabVIEW project demonstrates how to use the OPC UA VIs to programmatically create an OPC UA server application, create an OPC UA client application, and establish a connection between an OPC UA server and OPC UA client for data exchange both on Windows targets and on LabVIEW Real-Time targets. This project contains the following VIs:

  • Server VI—Use the Server VI to create an OPC UA server.
  • Client VI—Use the Client VI to create an OPC UA client and establish a connection between the OPC UA server and the OPC UA client.

Requirements

  • LabVIEW Datalogging and Supervisory Control Module or the LabVIEW Real-Time Module
  • (Real-Time Module) RT target

Instructions

You can run this project when an OPC UA server and OPC UA client are on the same machine or on different machines.

Running this Project When an OPC UA Server and OPC UA Client Are on the Same Machine

Complete the following steps to run this project.

  1. (Real-Time Module) Complete the following steps to specify an RT target.

    Note You must ensure that OPC UA is installed on the RT target that you specify.

    1. In the Project Explorer window, right-click RT target and select Properties from the shortcut menu to launch the RT Target Properties dialog box.
    2. On the General page of the Category list, specify the IP address for the RT target in the IP Address component.
  2. Run the Server VI to create an OPC UA server.
  3. Run the Client VI to create an OPC UA client and establish a connection between the OPC UA server and the OPC UA client.

Running this Project When an OPC UA Server and OPC UA Client Are on Different Machines

Complete the following steps to use this project.

  1. (Real-Time Module) Complete the following steps to specify an RT target.

    Note You must ensure that the OPC UA is installed on the RT target that you specify.

    1. In the Project Explorer window, right-click RT target and select Properties from the shortcut menu to launch the RT Target Properties dialog box.
    2. On the General page of the Category list, specify the IP address for the RT target in the IP Address component.
  2. Run the Server VI and the Client VI to create certificate files. The certificate files that the Server VI and the Client VI create reside in the default directory. Refer to the LabVIEW Help for more information about the default certificate file directory.
  3. Copy and paste the Server Demo Certificate.der file from the default directory on the machine where the OPC UA server resides into a location on the machine where the OPC UA client resides. Rename the Server Demo Certificate.der file.
  4. Copy and paste the Client Demo Certificate.der file from the default directory on the machine where the OPC UA client resides into a location on the machine where the OPC UA server resides. Rename the Client Demo Certificate.der file.
  5. Stop the Server VI and the Client VI.
  6. On the block diagram of the Server VI, add an element in the Build Array VI and enter the file path of the certificate file you copied and renamed to the server.
  7. On the block diagram of the Client VI, add an element in the Build Array VI and enter the file path of the certificate file you copied and renamed to the client.
  8. Run the Server VI to create an OPC UA server.
  9. Run the Client VI to create an OPC UA client and establish a connection between the OPC UA server and the OPC UA client.
